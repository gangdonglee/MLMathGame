# 8장. 정보이론 — 엔트로피, KL, cross-entropy

> "확률분포 간의 거리" 를 재는 도구가 KL divergence. 분류 NN 의 손실 (cross-entropy), 변분 추론 (ELBO), 강화학습의 정책 정규화 (PPO 의 KL 패널티), NES 의 자연 그래디언트 — 모두 이 장의 직접 응용.

---

## 8.0 한 장 요약

- **엔트로피** $H(p) = -\sum p \log p$ : 분포 $p$ 의 *불확실성* 의 정량.
- **KL divergence** $D_{KL}(p \| q) = \sum p \log (p/q) \ge 0$, $= 0 \iff p = q$. 분포 사이의 *비대칭 거리*.
- **Cross-entropy** $H(p, q) = -\sum p \log q = H(p) + D_{KL}(p \| q)$. 분류 NN 의 표준 손실.
- 분류 손실의 최소화 = 모델 분포가 데이터 분포에 *가까이 가는* 것 = MLE.
- ES 의 argmax 정책은 entropy = 0 의 deterministic 정책. 확률적 정책 (softmax) 으로 가면 entropy regularization, KL trust region 으로 자연 확장.

---

## 8.1 엔트로피

### 정의 8.1 (Shannon 엔트로피)

이산 확률분포 $p$ 위에서

$$
H(p) := -\sum_{x} p(x) \log p(x).
$$

(밑은 보통 $e$ 또는 2. 본 교재는 $e$, 단위는 nats.)

### 직관

- $p$ 가 *결정론적* (한 점에 집중) → $H = 0$ : 불확실성 0.
- $p$ 가 *균등* ($K$ 개 사건에 1/K) → $H = \log K$ : 최대 불확실성.

### 정리 8.2 (엔트로피의 비음성과 상한)

$0 \le H(p) \le \log K$ ($K$ = $p$ 의 support 크기). 등호: 왼쪽은 $p$ 가 한 점에 집중, 오른쪽은 균등.

**증명 (상한).** Jensen (정리 6.7) 으로 $H(p) = E_p[-\log p(X)] = E_p[\log (1/p(X))] \le \log E_p[1/p(X)] = \log K$. □

### 결합 / 조건부 엔트로피

- $H(X, Y) := -\sum p(x, y) \log p(x, y)$
- $H(X | Y) := H(X, Y) - H(Y) = E_Y[H(X | Y = y)]$

체인룰: $H(X, Y) = H(X) + H(Y | X)$.

### 미분 엔트로피 (연속)

$h(X) = -\int f(x) \log f(x) dx$. 음수일 수도 있어 *분포 사이의 차* 만 의미 가짐. KL 은 연속에서도 잘 정의 (음수 안 됨).

---

## 8.2 KL divergence

### 정의 8.3

확률분포 $p$, $q$ ($p$ 의 support $\subseteq q$ 의 support) 에 대해

$$
D_{KL}(p \| q) := \sum_x p(x) \log \frac{p(x)}{q(x)} = E_p\left[ \log \frac{p(X)}{q(X)} \right].
$$

연속도 동일 (적분).

### 정리 8.4 (Gibbs 부등식)

$D_{KL}(p \| q) \ge 0$, 등호 ⟺ $p = q$ a.e.

**증명.** Jensen ($-\log$ 는 볼록):

$$
D_{KL}(p \| q) = E_p\left[-\log \frac{q(X)}{p(X)}\right] \ge -\log E_p\left[\frac{q(X)}{p(X)}\right] = -\log \sum_x q(x) = -\log 1 = 0.
$$

등호 조건은 Jensen 의 등호 조건 = $q(X)/p(X)$ 가 상수 = $p = q$. □

### 비대칭

$D_{KL}(p \| q) \ne D_{KL}(q \| p)$ 일반적으로. ML 에서는 *목적에 따라* 어느 쪽인지 신중히 선택.

| 방향 | 의미 |
|---|---|
| $D_{KL}(p \| q)$ ($p$ = data, $q$ = model) | "data 가 있는 곳을 model 이 cover 해야 함" — mode-covering |
| $D_{KL}(q \| p)$ | "model 이 data 가 있는 곳에만 있어야" — mode-seeking. VAE/GAN 등에서 등장 |

### 정리 8.5 (체인룰)

$$
D_{KL}(p(x, y) \| q(x, y)) = D_{KL}(p(x) \| q(x)) + E_{p(x)}[D_{KL}(p(y|x) \| q(y|x))].
$$

ELBO, 변분 추론의 핵심 도구.

## 8.3 Cross-entropy

### 정의 8.6

$$
H(p, q) := -\sum_x p(x) \log q(x).
$$

### 정리 8.7 (분해)

$$
H(p, q) = H(p) + D_{KL}(p \| q).
$$

**증명.** $H(p, q) = -\sum p \log q = -\sum p \log p + \sum p \log (p/q) = H(p) + D_{KL}(p \| q)$. □

### 의미

$p$ 가 데이터의 진짜 분포 (학습 중 고정), $q_\theta$ 가 모델 분포 (학습 대상).

- $H(p)$ 는 $\theta$ 에 무관 (상수).
- $\arg\min_\theta H(p, q_\theta) = \arg\min_\theta D_{KL}(p \| q_\theta)$.

따라서 **cross-entropy 최소화 = KL 최소화 = 모델이 데이터 분포에 수렴**.

### 정리 8.8 (cross-entropy = log-likelihood 의 음수)

데이터 $\{x_i\}_{i=1}^n \overset{iid}{\sim} p$. 경험분포 $\hat p(x) = \tfrac{1}{n} \sum \mathbb{1}[x_i = x]$. 그러면

$$
H(\hat p, q_\theta) = -\tfrac{1}{n} \sum_{i=1}^n \log q_\theta(x_i) = -\tfrac{1}{n} \log \mathcal{L}(\theta).
$$

즉 cross-entropy 최소화 = MLE (정의 7.7).

이 동치성이 NN 분류 학습의 표준 손실이 cross-entropy 인 *수학적* 이유.

## 8.4 분류 NN 의 손실

### Softmax 모델

$K$-클래스 분류. NN 출력 logits $z = f_\theta(x) \in \mathbb{R}^K$. softmax 변환

$$
q_\theta(y = k | x) = \frac{e^{z_k}}{\sum_j e^{z_j}}.
$$

### Cross-entropy 손실

데이터 $(x_i, y_i)$, $y_i \in \{1, \dots, K\}$. one-hot $\delta_{y_i}$. 손실

$$
\mathcal{L}(\theta) = -\sum_i \log q_\theta(y_i | x_i) = -\sum_i \log \frac{e^{z_{y_i}^{(i)}}}{\sum_j e^{z_j^{(i)}}}.
$$

이것이 PyTorch/TensorFlow 의 `CrossEntropyLoss` 한 줄 안의 전부.

### 그래디언트 — 짧고 우아

$$
\frac{\partial \mathcal{L}}{\partial z_k^{(i)}} = q_\theta(k | x_i) - \delta_{y_i = k}.
$$

**증명 스케치.** $\log q = z_y - \log \sum e^{z_j}$. $\partial / \partial z_k$ 직접 계산. $\partial \log \sum e^{z_j} / \partial z_k = e^{z_k} / \sum e^{z_j} = q(k)$. 모음. □

이 깔끔함이 cross-entropy + softmax 가 짝으로 쓰이는 또 다른 이유. 백프롭 시 출력층 그래디언트가 *예측 - 정답* 형태.

## 8.5 정책 정규화 — KL 의 RL 응용

### 문제

강화학습에서 정책 $\pi_\theta(a | s)$ 를 점진적으로 개선할 때, *너무 큰 변화* 는 학습 붕괴. 새 정책이 옛 정책에서 *너무 멀리* 가지 못하게 막고 싶다.

### KL trust region

다음 update 의 정책 $\pi_{\theta'}$ 가 $\pi_\theta$ 에서 KL 거리로 가깝도록 제약:

$$
\max_{\theta'} \mathcal{J}(\theta') \quad \text{s.t.} \quad E_s[D_{KL}(\pi_\theta(\cdot | s) \| \pi_{\theta'}(\cdot | s))] \le \delta.
$$

이것이 **TRPO** 의 핵심. PPO 는 KL 을 명시적 제약 대신 *clipping* 으로 근사.

### 본 프로젝트와의 관계

TinyCommand 의 정책은 *deterministic argmax*. 확률적 정책으로 확장하면 (예: softmax over 6 actions, temperature τ) 위 KL 정규화가 자연스럽게 적용 가능. 현재는 deterministic 이라 KL 은 0/∞ 의 두 값만 가지므로 의미가 약하다.

## 8.6 자연 그래디언트와 NES (preview)

### Fisher 정보 행렬

확률분포 모델 $p(x | \theta)$ 의 Fisher 정보

$$
\mathcal{I}(\theta) := E_{p_\theta}\left[ \nabla_\theta \log p(X | \theta) \nabla_\theta \log p(X | \theta)^\top \right] \in \mathbb{R}^{n \times n}.
$$

대칭 PSD. 작은 $\theta \to \theta + \Delta\theta$ 에 대해 $D_{KL}(p_\theta \| p_{\theta + \Delta\theta}) \approx \tfrac{1}{2} \Delta\theta^\top \mathcal{I} \Delta\theta$. 즉 *Fisher 가 KL 의 국소 metric*.

### 자연 그래디언트

GD 의 update $\Delta\theta = -\alpha \nabla \mathcal L$ 대신

$$
\Delta\theta = -\alpha \mathcal{I}^{-1} \nabla \mathcal L.
$$

이것이 **natural gradient**. 모수의 좌표계가 바뀌어도 *같은 분포 변화* 를 만든다.

### NES = ES 의 natural gradient

NES (Natural Evolution Strategy) 는 정책 모수가 아니라 *탐색 분포* $p(\theta | \phi) = \mathcal{N}(\mu, \Sigma)$ 의 모수 $\phi = (\mu, \Sigma)$ 에 대해 natural gradient 를 적용. 결과적으로

- $\mu$ update 는 OpenAI ES 식 (10장).
- $\Sigma$ update 는 *공분산을 학습 방향으로 적응* (3장 SVD 의 응용).

본 프로젝트는 $\Sigma = \sigma^2 I$ 고정 → $\mu$ 만 업데이트하는 가장 단순한 NES 의 특수형.

## 8.7 Mutual information

### 정의 8.9

$$
I(X; Y) := D_{KL}(p(x, y) \| p(x) p(y)) = H(X) - H(X | Y).
$$

"$Y$ 를 알면 $X$ 의 불확실성이 얼마나 줄어드나". 0 ⟺ 독립.

### 응용

- **InfoGAN, MINE**: 표현 학습에서 $X$ 와 잠재변수 $Z$ 의 MI 를 최대화.
- **feature selection**: $I(X_i; Y)$ 가 작은 feature 를 제거.
- **information bottleneck**: $I(X; T) - \beta I(T; Y)$ 의 trade-off.

본 프로젝트는 직접 사용 안 함.

## 8.9 TinyCommand 연결

### 8.9.1 argmax 정책 = 0 엔트로피

[PolicyEngine.cpp](../../ML/AI/PolicyEngine.cpp) 의 `Tick()` 이 6 행동 점수를 받아 `argmax` 한다. 이는 deterministic 정책 ⇒ $H(\pi) = 0$. 모든 확률 mass 가 한 점에 집중.

### 8.9.2 softmax 로 전환하면

만약 `Tick()` 을 다음으로 바꾸면:

```cpp
float probs[6];
softmax(scores, probs, 6);  // p_k = exp(z_k) / Σ exp(z_j)
int action = sample_categorical(probs);
```

정책이 확률적이 되어 entropy > 0. ES 의 점수 $J$ 자체가 노이즈를 가지므로 추가 entropy regularization $\mathcal L = -J + \lambda H(\pi)$ 같은 식으로 *탐색 다양성* 을 보장 가능.

### 8.9.3 cross-entropy 가 안 보이는 이유

본 프로젝트는 *labeled action* 데이터가 없다. 만약 사람이 "이 상황 → 이 행동" 을 라벨링한 dataset 이 있다면 cross-entropy supervised pretraining 가능. 그 후 ES fine-tune. (5장 §5.8.2 의 hybrid.)

### 8.9.4 NES 로 가는 길 — Fisher 의 의미

현 ES 는 $\Sigma = \sigma^2 I$ 고정. $\sigma$ 를 hand-tune 한다 (`MLTrainer::Params::sigma = 0.10f`). Fisher 정보를 계산해 자동 적응하면 *학습이 빠르고 안정* 하지만 구현 비용이 크다 (1670 × 1670 행렬 추정 + 역). 본 프로젝트는 단순함을 선택.

### 8.9.5 KL 로 본 학습 진행

세대마다 $\theta_t$ 가 만드는 노이즈 분포 $\mathcal{N}(\theta_t, \sigma^2 I)$ 의 KL 변화

$$
D_{KL}(\mathcal{N}(\theta_t, \sigma^2 I) \| \mathcal{N}(\theta_{t+1}, \sigma^2 I)) = \frac{\|\theta_{t+1} - \theta_t\|^2}{2 \sigma^2}.
$$

(평균만 다른 등방 정규의 KL 닫힌 형식.) 즉 *한 step 의 KL = update 노름의 제곱 / 2σ²*. 학습 진행을 KL 단위로 측정 가능 (PPO 가 정확히 이 양을 모니터).

## 8.10 연습문제

1. $K$-사건 균등분포의 엔트로피 $H = \log K$ 를 정의 8.1 로 직접 계산하라.
2. $p = (0.5, 0.5)$, $q = (0.9, 0.1)$ 의 $D_{KL}(p\|q)$ 와 $D_{KL}(q\|p)$ 를 계산해 비대칭을 확인.
3. 정리 8.4 (Gibbs) 의 등호 조건 ($p = q$ a.e.) 을 Jensen 의 strict convexity 로 정당화하라.
4. softmax cross-entropy 의 출력층 그래디언트 $\partial \mathcal L / \partial z_k = q(k) - \delta_{y=k}$ 를 직접 유도하라.
5. 두 등방 정규 $p = \mathcal{N}(\mu_1, \sigma^2 I)$, $q = \mathcal{N}(\mu_2, \sigma^2 I)$ 의 KL 닫힌 형식을 유도하라.
6. (TinyCommand) `PolicyEngine::Tick` 을 softmax 샘플링으로 바꿨을 때, ES 학습 신호의 분산이 어떻게 변하는지 정성적으로 (정리 6.15 와의 상호작용 포함) 예측하라.
